Разработка Аис

Техническое задание

1. Разработка диаграммы вариантов использования, диаграммы классов, диаграммы последовательностей, диаграммы кооперации, диаграммы состояний, диаграммы деятельности (MS Visio, Rational Rose и другие) – в процессе.

2. На основании анализа предметной области сформировать «Техническое задание», оформить в соответствии с требованиями стандартов ЕСПД:

  • 1.1. Наименование системы: Система управления телефонными номерами и тарифами
  • 1.2. Основание для разработки: Анализ предметной области телефонного обмена

2. Цель разработки:
Целью разработки системы является создание автоматизированной программы для управления абонентскими данными, телефонными номерами, тарифами и информацией о вызовах для улучшения обработки данных, управления клиентами и анализа долгов.

3. Область применения:
Система будет использоваться для управления данными абонентов, обработки информации о звонках, управления тарифами и отслеживания балансов абонентов.

4. Требования к системе:

  • 4.1. Функциональные требования:
    • Управление данными абонентов (создание, чтение, обновление и удаление записей)
    • Управление тарифными планами и ценами
    • Просмотр и обработка данных о звонках
    • Управление балансом абонентов и статусом (активен/заблокирован)
    • Функция авторизации и аутентификации пользователей системы
  • 4.2. Нефункциональные требования:
    • Интерфейс должен быть интуитивно понятным и удобным для пользователей
    • Система должна обеспечивать защиту данных и безопасную аутентификацию пользователей
    • Высокая скорость обработки запросов к базе данных

5. Требования к надежности:

  • 5.1. Защита данных: система должна обеспечивать резервное копирование данных и защиту от несанкционированного доступа.
  • 5.2. Отказоустойчивость: система должна корректно обрабатывать ошибки и не прерывать работу при сбоях.

6. Условия эксплуатации:
Система должна работать на Windows и быть доступна через приложение, написанное на Java, с использованием JavaFX для графического интерфейса и PostgreSQL для управления базой данных.

7. Требования к программному обеспечению:

  • 7.1. Язык программирования: Java, 1С
  • 7.2. Среда разработки: IntelliJ IDEA, 1C предприятия
  • 7.3. Среда выполнения: Windows 7/10/11
  • 7.4. База данных: PostgreSQL

8. Требования к документированию:
К системе должны быть предоставлены следующие документы:

  • Руководство пользователя
  • Руководство администратора
  • Техническое руководство по установке и настройке системы

  • 9.2. Проектирование системы и базы данных

  • 9.4. Тестирование системы и исправление ошибок
  • 9.5. Внедрение и обучение персонала

10. Порядок контроля и приемки:

  • 10.1. Приемка: система будет считаться принятой, если она соответствует всем требованиям ТЗ и успешно проходит все этапы тестирования.

Краткая спецификация разрабатываемой системы

1. Разработка диаграммы вариантов использования, диаграммы классов, диаграммы последовательностей, диаграммы кооперации, диаграммы состояний, диаграммы деятельности (MS Visio, Rational Rose и другие) – в процессе.

2. На основании анализа предметной области сформировать «Техническое задание», оформить в соответствии с требованиями стандартов ЕСПД:

1. Назначение системы
Система предназначена для управления данными телефонных абонентов, включая информацию о звонках, клиентах, их тарифах и состоянии счетов. Система позволяет просматривать и добавлять данные абонентов, отслеживать задолженности и историю звонков, а также проводить авторизацию пользователей для управления доступом к функциональности системы.

2. Функциональные требования

  • 2.1 Авторизация пользователя - Проверка введенных данных (логин и пароль) пользователя в базе данных. В случае успешной авторизации отображение главного окна с доступом к функциональности системы.
  • 2.2 Управление данными абонентов
    • Просмотр и добавление новых записей о клиентах, включая ФИО, адрес, номера телефонов, модель телефона, тарифный план.
    • Добавление записей о звонках с указанием входящего и исходящего абонентов и номеров телефонов.
    • Просмотр и добавление информации о тарифах, стоимости и типах тарифов.
  • 2.3 Управление счетами абонентов
    • Отображение списка абонентов с задолженностью, включая баланс и статус счета.
    • Просмотр списка разблокированных абонентов без задолженности.

3. Нефункциональные требования

  • 3.1 Надежность и отказоустойчивость - Обработка ошибок подключения к базе данных и некорректного ввода данных.
  • 3.2 Удобство использования - Простой и интуитивно понятный интерфейс на основе JavaFX.
  • 3.3 Производительность - Минимизация задержек при выполнении SQL-запросов и отображении данных.
  • 3.4 Масштабируемость - Возможность расширения функциональности, добавление новых категорий данных и таблиц при необходимости.

4. Архитектура системы
Система построена на базе клиент-серверной архитектуры, где клиентская часть реализована с использованием JavaFX, а серверная часть представлена базой данных (например, PostgreSQL) с использованием JDBC для соединения. Основные компоненты системы включают:

  • Модуль данных: классы Call, ClientNumber, User, Tariff для хранения информации о звонках, абонентах и тарифах.
  • Контроллеры: DashboardController и LoginController для обработки запросов пользователя и взаимодействия с базой данных.
  • База данных: хранение информации о пользователях, звонках, счетах и тарифах.

5. Пользовательский интерфейс

  • Форма авторизации: Поля для ввода логина и пароля, сообщение об ошибке в случае неудачной авторизации.
  • Главное окно (Dashboard): Таблицы для отображения информации о звонках, номерах клиентов, тарифах и задолженностях. Кнопки для выполнения фильтрации и добавления новых данных.

6. Интеграционные требования
Подключение к базе данных: использование JDBC для соединения с PostgreSQL. Поддержка взаимодействия с FXML: загрузка и отображение интерфейсов, разработанных в формате FXML.

7. Ограничения
Приложение предназначено для работы с конкретной базой данных, поддержка других баз данных потребует доработки кода. Поддержка работы только на устройствах с установленной Java Virtual Machine (JVM).

2.5 Руководство пользователя

Это руководство пользователя предназначено для системы управления телефонной сетью. В нем описаны основные функции приложения и способы их использования.

2. Основные функции

2.1 Добавление нового пользователя

Для добавления нового пользователя выполните следующие действия:

  • Нажмите кнопку "Добавить пользователя" в главном меню.
  • Заполните форму с данными пользователя.
  • Нажмите кнопку "Сохранить", чтобы добавить пользователя в систему.
  • После успешного добавления появится сообщение об успешном добавлении.

3. Часто задаваемые вопросы (FAQ)

Вопрос: Как добавить пользователя в систему?
Ответ: Нажмите "Добавить пользователя", заполните данные и сохраните их.

Вопрос: Почему я не вижу список клиентов с долгами?
Ответ: Возможно, на текущий момент все клиенты погасили свои задолженности.
